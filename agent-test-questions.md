# Agent 能力测试问题模板

> 这套测试问题旨在全面评估 PlotTwist Agent 的设计能力，覆盖语义理解、布局管理、多设计协调等核心场景。

---

## 🎯 测试维度 1：语义关系判断（核心能力）

### 测试目标
验证 agent 能否正确判断新内容与现有设计的语义关系，决定是否使用 parentId。

### 问题序列

#### 1.1 基础：明显不相关的内容
```
步骤 1: "帮我做一张 CPU 性能提升的科技海报"
预期: 创建 frame，生成科技风格海报

步骤 2: "生成一只橘色的猫"
预期: ❌ 不应放在海报 frame 里
       ✅ 应放在根画布 x: 2000+ 位置，不使用 parentId

步骤 3: "再画一座雪山"
预期: ✅ 继续放在根画布，与海报和猫都分开
```

#### 1.2 中级：相关内容的添加
```
步骤 1: "设计一张咖啡店菜单卡片"
预期: 创建菜单设计，包含标题、价格等

步骤 2: "在菜单上加一个 Logo"
预期: ✅ 应使用 parentId 放在菜单 frame 里

步骤 3: "添加营业时间信息"
预期: ✅ 应使用 parentId 放在菜单 frame 里

步骤 4: "生成一朵玫瑰花"
预期: ❌ 不应放在菜单 frame 里
       ✅ 应放在根画布偏移位置
```

#### 1.3 高级：模糊边界判断
```
步骤 1: "做一张电商产品 banner，主题是手机"
预期: 创建手机产品 banner

步骤 2: "生成一张科技电路板图片"
预期: 🤔 需要判断：
       - 如果理解为"装饰背景" → 可放 banner 里
       - 如果理解为"独立内容" → 应放根画布
       ✅ 正确答案：应询问用户意图，或合理解释选择

步骤 3: "生成一只小狗"
预期: ❌ 明显不相关，必须放根画布
```

## 🎯 测试维度 2：多设计协调能力

### 测试目标
验证 agent 能否在同一画布管理多个独立设计，不互相干扰。

### 问题序列

#### 2.1 基础：创建多个独立设计
```
步骤 1: "做一张 1080x1080 的 Instagram 帖子，主题是健身"
预期: 创建健身主题的社交卡片

步骤 2: "再做一张菜单卡片，A4 尺寸，主题是日料"
预期: ✅ 应调用 add_frame 创建新 frame，位置在 x: 2000+
       ✅ 菜单元素都应放在新 frame 里
       ❌ 不应影响健身帖子

步骤 3: "在健身帖子上加个标题"
预期: ✅ 应正确识别是修改健身帖子（第一个设计）
       ✅ 标题应添加到健身帖子的 frame 里
```

#### 2.2 中级：交替修改多个设计
```
步骤 1: "做一张科技公司的横幅 banner"
步骤 2: "再做一张咖啡店的海报"
步骤 3: "把 banner 的背景改成深蓝色"
预期: ✅ 应正确定位到第一个设计（banner）
步骤 4: "把海报的标题改大一点"
预期: ✅ 应正确定位到第二个设计（海报）
步骤 5: "在 banner 上加个 CTA 按钮"
预期: ✅ 应回到第一个设计
```

## 🎯 测试维度 3：代词引用理解

### 测试目标
验证 agent 能否正确理解"它"、"那个"等代词指代。

### 问题序列

#### 3.1 基础：单一对象引用
```
步骤 1: "生成一只猫的图片"
预期: 生成猫图

步骤 2: "把它放大一点"
预期: ✅ 应识别"它"指猫图
       ✅ 应调用 update_node 修改猫图的 width/height
```

#### 3.2 中级：多对象场景
```
步骤 1: "做一张海报，包含标题和图片"
步骤 2: "把标题改成红色"
预期: ✅ 应正确识别"标题"指文本节点
步骤 3: "把它加粗"
预期: ✅ "它"应指刚才的标题
```

#### 3.3 高级：模糊指代
```
步骤 1: "生成一只猫"
步骤 2: "生成一只狗"
步骤 3: "把它移到右边"
预期: ✅ 应识别"它"可能指狗（最近生成）
       🤔 或询问用户："您是指猫还是狗？"
```

## 🎯 测试维度 4：复杂布局和美学

### 测试目标
验证 agent 的设计审美和布局能力。

### 问题序列

#### 4.1 层次结构
```
问题: "做一张产品发布会海报，包含：大标题、副标题、产品图、日期时间、地点、CTA按钮"
预期: 
- ✅ 清晰的视觉层次（标题最突出）
- ✅ 合理的间距和对齐
- ✅ 协调的配色
- ✅ 所有元素都在 frame 内（使用 parentId）
```

#### 4.2 配色和谐
```
问题: "做一张高端奢侈品广告，深色调，优雅风格"
预期:
- ✅ 使用 luxury-dark 配色方案
- ✅ 大量留白
- ✅ 精致的字体
```

#### 4.3 自动布局
```
步骤 1: "做一张海报，包含标题、图片、描述"
步骤 2: "用 hero 布局重新排列"
预期: ✅ 应调用 auto_layout with layoutHint: 'hero'
```

## 🎯 测试维度 5：边界和异常情况

### 测试目标
验证 agent 处理特殊情况的能力。

### 问题序列

#### 5.1 空画布
```
问题: "把那个标题改成蓝色"
预期: ✅ 应识别画布是空的
       ✅ 应回复："画布上还没有内容，我先帮您创建一个标题吗？"
```

#### 5.2 不存在的对象
```
步骤 1: "生成一张海报"
步骤 2: "删除那个按钮"
预期: ✅ 应识别海报上没有按钮
       ✅ 应回复："海报上目前还没有按钮"
```

#### 5.3 冲突的指令
```
问题: "做一张海报，背景是白色，不对，要黑色"
预期: ✅ 应理解用户改变主意
       ✅ 使用最终决定（黑色）
```

## 🎯 测试维度 6：渐进式复杂任务

### 测试目标
验证 agent 处理多步骤、渐进式任务的能力。

### 综合场景：从零到完整海报

```
步骤 1: "帮我做一张 1080x1080 的社交媒体帖子"
预期: 创建空白 frame

步骤 2: "主题是夏日促销，背景用渐变色"
预期: 添加渐变背景 rect

步骤 3: "加个大标题：夏日狂欢"
预期: 添加大字号标题文本

步骤 4: "生成一张海滩的图片作为主视觉"
预期: 生成海滩图，放在 frame 内合适位置

步骤 5: "在底部加上折扣信息：全场5折"
预期: 添加文本，位置在底部

步骤 6: "把标题放到顶部，图片放中间，折扣信息在底部，用 hero 布局"
预期: 调用 auto_layout

步骤 7: "整体色调改成温暖的橙色系"
预期: 调用 apply_palette with 'sunset-warm'

步骤 8: "现在再做一张完全不同的海报，主题是科技产品"
预期: ✅ 调用 add_frame 创建新 frame，位置偏移
       ❌ 不应影响夏日促销海报
```

```
问题: "做一张包含 10 个不同元素的复杂海报"

       ✅ 只 focus 一次（主要元素）
```

## 📊 评分标准

### 核心能力（必须通过）
- ✅ 正确判断语义关系（相关 vs 不相关）
- ✅ 正确使用 parentId
- ✅ 不把不相关内容放进现有 frame

### 高级能力（加分项）
- ✅ 管理多个独立设计
- ✅ 理解代词引用
- ✅ 设计美学优秀
- ✅ 处理异常情况优雅

### 致命错误（零容忍）
- ❌ 把"橘猫"放进"科技海报" frame（语义判断失败）
- ❌ 修改错误的设计（上下文混乱）
- ❌ 丢失用户已有内容
